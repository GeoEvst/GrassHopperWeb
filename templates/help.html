В данном проекте представлено шифрование и расшифрование по криптографическому алгоритму "Кузнечик"
На вкладке "Зашифровать" представлено 3 режима шифрования.

ГОСТ 34.12.2015

Данный режим демонстрирует работу "чистого" алгоритма из ГОСТ. Кузнечик — блочный алгоритм, он работает с блоками данных длинной 128 бит или 16 байт. Длина ключа составляет 256 бит (32 байта). Сообщение, подлежащее шифрованию и ключ должны быть как в ГОСТ. В данном режиме можно ознакомиться с работой шифра и проверить корректность его работы. Для этого можно подставить ключ и блок данных из контрольного примера ГОСТ 34.12.2015. На вкладке "Зашифровать" нужно выбрать режим "ГОСТ 34.12.2015", раскрыть "формат записи данных", там будут указаны контрольные примеры ключа и блока данных. Или можно самостоятельно скопировать данные значения из ГОСТ, далее нужно нажать "зашифровать", отобразиться зашифрованное сообщение.

В данном режиме можно "Сгенерировать ключ", ключ будет рандомно сгенерирован, далее можно ввести свой блок данных согласно формату и передать получателю ключ и зашифрованное сообщение для его расшифровки.
Если есть необходимость зашифровать несколько блоков данных при использовании одного ключа, то в данном режиме, с точки зрения криптографии, это делать небезопасно. Для этого следует воспользоваться режимом ГОСТ 34.13.2015.

ГОСТ 34.13.2015

Данный режим также демонстрирует работу "чистого" алгоритма из ГОСТ, но в данном случае демонстрируется ГОСТ 34.13.2015. Данный режим, в отличие от предыдущего, позволяет безопасно шифровать более одного блока данных одним ключом шифрования. В данном режиме можно ознакомиться с работой шифра и проверить корректность его работы. На вкладке "Зашифровать" нужно выбрать режим "ГОСТ 34.13.2015", раскрыть "формат записи данных", там будут указаны контрольные примеры ключа, синхропосылки и блока данных. Или можно самостоятельно скопировать данные значения из ГОСТ, далее нужно нажать "зашифровать", отобразиться зашифрованное сообщение. Если исходное сообщение не будет кратно длине одного блока данных (128 бит) то последний блок будет автоматически дополнен нулями.

Зашифровать текстовое сообщение

Данный режим предоставляет возможность шифровать обыкновенный текст (латиница/киррилица). Так как текстовое сообщение может быть достаточно длинным, шифрование производится по ГОСТ 34.13.2015 в режиме гаммирования, но следует учитывать, для безопасности следует использовать новый ключ для каждого последующего текстового сообщения. Каждый символ из текста преобразовывается в двоичный код, согласно кодировке utf-8, далее данный код преобразовывается в 16-ричный формат, из всех преобразованных символов формируются 16-ричные блоки данных. Далее эти блоки шифруются по ГОСТ 34.13.2015. Для шифрования в данном режиме следует ввести ключ шифрования, синхропосылку (гамму шифра), либо сгенерировать рандомно данные значения, далее ввести сообщение и нажать "зашифровать".
Для проверки можно вставить значения ключа и синхропосылки из ГОСТ, данные значения представлены в формате записи, или придумать данные значения самостоятельно, ввести любой текст, зашифровать.

На вкладке "Расшифровать" в поле ввода ключа и синхропосылки вставить значения, которые использовались для шифрования, ввести шифрованное сообщение и нажать "расшифровать".

Для удобства, если использовались значения ключа и синхропосылки из ГОСТ, то при расшифровании можно нажать на кнопки около полей ввода и подставить значения из ГОСТ, без необходимости их копировать.

Подробнее о работе шифра можно прочитать в разделе "О шифре".
